# 에이전트 화면 UI/UX 개선 계획

## 📊 현재 문제점 분석

### 1. 레이아웃 문제점

#### 1.1 세로 스크롤 과다
**발견 위치**: `src/pages/main/Agent.tsx`

**문제점**:
- 모든 컴포넌트가 세로로 쌓여있어 화면이 매우 길어짐
- 사용자가 결과를 보기 위해 많은 스크롤이 필요함
- 한 화면에 보이는 정보량이 적어 효율성이 떨어짐

**현재 구조**:
```
┌─────────────────────────┐
│ Hero 섹션 (설명)         │
├─────────────────────────┤
│ MenuRecommendation      │
│ (메뉴 추천 받기)         │
├─────────────────────────┤
│ RestaurantList          │
│ (일반 검색 결과)         │
├─────────────────────────┤
│ AiPlaceRecommendations  │
│ (AI 추천 결과)          │
└─────────────────────────┘
```

**문제 시나리오**:
1. 사용자가 메뉴 추천을 받음
2. 메뉴 카드를 클릭하여 검색/AI 추천 실행
3. 결과가 아래에 나타나지만 화면 밖에 있어 스크롤 필요
4. 여러 메뉴를 비교하려면 계속 스크롤해야 함

#### 1.2 공간 활용 비효율
**문제점**:
- 데스크톱 화면에서 가로 공간을 충분히 활용하지 못함
- 모바일과 동일한 세로 레이아웃으로 데스크톱 UX 저하
- 결과를 비교하거나 여러 정보를 동시에 보기 어려움

#### 1.3 정보 계층 구조 불명확
**문제점**:
- 메뉴 추천, 일반 검색, AI 추천이 모두 동일한 레벨로 표시됨
- 어떤 것이 주요 액션인지 불명확
- 사용자 플로우가 직관적이지 않음

### 2. 사용자 경험 문제점

#### 2.1 결과 확인을 위한 스크롤 필요
**문제점**:
- 메뉴를 선택하면 결과가 아래에 나타나지만 화면 밖에 위치
- 자동 스크롤이 있지만 사용자가 컨텍스트를 잃을 수 있음
- 결과를 보기 위해 위로 올라가야 하는 경우 발생

#### 2.2 여러 메뉴 비교 어려움
**문제점**:
- 여러 메뉴의 추천 결과를 비교하려면 스크롤을 반복해야 함
- 이전 결과를 보려면 위로 스크롤해야 함
- 탭이나 사이드바 같은 네비게이션 부재

#### 2.3 로딩 상태 시각적 피드백 부족
**문제점**:
- 로딩 중에도 기존 콘텐츠가 그대로 보여 혼란 가능
- 로딩 위치가 명확하지 않음
- 진행 상태를 파악하기 어려움

## 🎯 개선 목표

### 1. 레이아웃 개선
- **데스크톱**: 2-3 컬럼 레이아웃으로 가로 공간 활용
- **모바일**: 기존 세로 레이아웃 유지하되 최적화
- **반응형**: 화면 크기에 따라 적절한 레이아웃 전환

### 2. 사용자 경험 개선
- **한 화면에 더 많은 정보 표시**: 스크롤 최소화
- **결과 비교 용이**: 여러 결과를 동시에 볼 수 있도록
- **명확한 정보 계층**: 주요 액션과 결과의 구분 명확화

### 3. 시각적 개선
- **카드 기반 레이아웃**: 각 섹션을 명확히 구분
- **애니메이션**: 부드러운 전환 효과
- **로딩 상태**: 명확한 피드백 제공

### 4. API 비용 절감 유지 (최우선)
- **상태 유지**: 다른 화면 이동 후 복귀 시에도 결과 유지
- **재사용**: 이미 로드된 결과는 Redux에서 읽어서 표시 (API 재호출 없음)
- **상태 관리 로직 보존**: UI 변경 시에도 Redux 상태 구조 및 관리 로직은 절대 변경하지 않음

## 📐 개선 방안

### Phase 1: 레이아웃 재구성

#### 1.1 데스크톱 레이아웃 (≥1024px)
**목표**: 2-3 컬럼 레이아웃으로 가로 공간 활용

**새로운 구조**:
```
┌─────────────────────────────────────────────────────┐
│ Hero 섹션 (상단, 전체 너비)                          │
├──────────────────────┬──────────────────────────────┤
│                      │                              │
│ 메뉴 추천 섹션       │  결과 영역                    │
│ (왼쪽, 고정)        │  (오른쪽, 동적)               │
│                      │                              │
│ - 입력 폼            │  - 일반 검색 결과             │
│ - 추천 메뉴 카드     │  - AI 추천 결과              │
│                      │  - 선택된 메뉴 표시          │
│                      │                              │
└──────────────────────┴──────────────────────────────┘
```

**구현 세부사항**:
- 왼쪽 컬럼 (30-40%): 메뉴 추천 섹션 고정
- 오른쪽 컬럼 (60-70%): 결과 표시 영역
- 결과 영역은 선택된 메뉴에 따라 동적으로 업데이트
- 스크롤은 각 컬럼별로 독립적으로 동작 (필요시)

#### 1.2 태블릿 레이아웃 (768px - 1023px)
**목표**: 2 컬럼 레이아웃 유지하되 비율 조정

**구조**:
- 왼쪽: 45%
- 오른쪽: 55%

#### 1.3 모바일 레이아웃 (<768px)
**목표**: 세로 레이아웃 유지하되 최적화

**구조**:
- Hero 섹션 (간소화)
- 메뉴 추천 섹션
- 결과 영역 (탭 또는 아코디언으로 전환)

### Phase 2: 컴포넌트 재구성

#### 2.1 메뉴 추천 섹션 개선
**위치**: 왼쪽 컬럼 (데스크톱)

**개선 사항**:
- 입력 폼을 더 컴팩트하게
- 추천 메뉴 카드를 그리드로 표시 (2열)
- 선택된 메뉴 하이라이트
- 스크롤 가능한 영역으로 제한 (최대 높이 설정)

**컴포넌트 구조**:
```tsx
<MenuRecommendationSection>
  <MenuRecommendationInput />  // 입력 폼
  <MenuRecommendationList />   // 추천 메뉴 카드들
  <LoadingIndicator />          // 로딩 상태
</MenuRecommendationSection>
```

#### 2.2 결과 영역 개선
**위치**: 오른쪽 컬럼 (데스크톱)

**개선 사항**:
- 선택된 메뉴 표시 (상단 고정)
- 탭 또는 섹션으로 일반 검색 / AI 추천 구분
- 각 결과를 카드 형태로 표시
- 빈 상태일 때 안내 메시지 표시

**컴포넌트 구조**:
```tsx
<ResultsSection>
  <SelectedMenuHeader />       // 선택된 메뉴 표시
  <ResultsTabs>                // 탭 네비게이션
    <Tab name="일반 검색">
      <RestaurantList />
    </Tab>
    <Tab name="AI 추천">
      <AiPlaceRecommendations />
    </Tab>
  </ResultsTabs>
  <EmptyState />               // 빈 상태
</ResultsSection>
```

#### 2.3 Hero 섹션 간소화
**개선 사항**:
- 높이 축소 (현재 p-8 → p-6)
- 텍스트 간격 최적화
- 로그인 안 한 사용자용 CTA 버튼 간소화

### Phase 3: 인터랙션 개선

#### 3.1 메뉴 선택 시 결과 영역 업데이트
**목표**: 선택 즉시 오른쪽 영역에 결과 표시

**구현**:
- 메뉴 클릭 시 확인 카드 모달 표시 (기존 유지)
- 선택 후 결과 영역으로 자동 포커스 (스크롤)
- 애니메이션으로 부드러운 전환

#### 3.2 탭 네비게이션
**목표**: 일반 검색과 AI 추천을 쉽게 전환

**구현**:
- 탭 UI로 두 결과 타입 전환
- 각 탭에 결과 개수 표시
- 활성 탭 하이라이트

#### 3.3 스크롤 최적화
**목표**: 필요한 영역만 스크롤

**구현**:
- 왼쪽 메뉴 섹션: 고정 높이 + 스크롤
- 오른쪽 결과 섹션: 동적 높이 + 스크롤
- 전체 페이지 스크롤 최소화

#### 문제 5: AI 추천 시 탭 자동 전환 미작동
**문제점**: 
- AI 추천을 받으면 오른쪽에서 바로 AI 추천 탭으로 옮겨지고 추천 로딩이나 결과를 보여줘야 하는데 일반 검색 화면에서 가만히 있음
- AI 추천 탭에서 추가로 다른 메뉴의 가게를 추천 받더라도 일반 검색 탭으로 가서 다시 AI 추천 탭을 확인해야 함
- **추가 발견**: 메뉴를 클릭해서 모달만 떴는데도 탭이 자동으로 변경되는 문제

**서비스 흐름 정리**:
1. **메뉴 선택 단계**: 왼쪽에서 메뉴 클릭 → 확인 카드 모달 표시
   - **이 단계에서는 탭 변경 없음** (아직 검색 방식 선택 전)
2. **검색 방식 선택 단계**: 모달에서 "일반 검색" 또는 "AI 추천" 선택
   - **이 단계에서 선택한 항목으로 탭 자동 전환**
   - 일반 검색 선택 → 일반 검색 탭으로 이동
   - AI 추천 선택 → AI 추천 탭으로 이동
3. **결과 표시 단계**: 선택한 탭에서 검색 결과 표시
   - AI 추천은 응답 시간이 길어서 다른 메뉴에 대해 또 검색할 수 있음
   - 이 경우에도 선택한 검색 방식에 맞는 탭 유지

**해결 방안**:
- **모달에서 선택하기 전에는 탭 변경 금지**
  - 메뉴 클릭만으로는 탭 변경하지 않음
  - `handleMenuClick`에서는 탭 변경하지 않음
- **모달에서 선택한 후에만 탭 자동 전환**
  - `handleSearch` 호출 시 → 일반 검색 탭으로 전환
  - `handleAiRecommendation` 호출 시 → AI 추천 탭으로 전환
- **AI 추천 완료 후에도 AI 추천 탭 유지**
  - AI 추천이 완료되어도 AI 추천 탭 유지
- **다른 메뉴 검색 시에도 선택한 검색 방식 탭 유지**
  - AI 추천 중에 다른 메뉴로 일반 검색 → 일반 검색 탭 유지
  - 일반 검색 중에 다른 메뉴로 AI 추천 → AI 추천 탭 유지

**구현 위치**: 
- `src/pages/main/Agent.tsx`: `handleSearch`, `handleAiRecommendation`에서 탭 전환
- `src/components/features/agent/ResultsSection.tsx`: 메뉴 선택만으로는 탭 변경하지 않도록 수정
