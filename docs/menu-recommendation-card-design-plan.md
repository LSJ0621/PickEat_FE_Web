# 메뉴 추천 카드 디자인 계획

## 1) 목표
- 가독성 높은 카드 레이아웃으로 추천 메뉴를 빠르게 스캔 가능하게 만든다.
- 선택 상태, 순서, AI 추천 맥락을 명확히 표현한다.
- 모바일~데스크톱 반응형에서 일관된 경험을 제공한다.

## 2) 현재 문제
- 가로 스크롤형 리스트라 한 화면 내 시각적 질서가 약함.
- 메뉴 정보가 텍스트 한 줄뿐이라 추천 성격(우선순위/AI 추천) 전달이 부족.
- 선택/비선택 상태의 대비가 다소 약함.

## 3) 레이아웃 가이드 (새 구조 전제)
- 그리드: 모바일 1열 → 태블릿 2열 → 데스크톱 3열, `max-height` 지정 후 세로 스크롤.
- 카드 높이: 92~108px 수준, 내부 여백 16~20px.
- 간격: 카드 간 12~16px, 섹션 상단 여백 12~16px.

## 4) 카드 구성
- 좌측 컬러 스트립: 랭크/순번 색인(세로 스트립), 숫자는 작은 사각 태그로 표기.
- 본문: 메뉴명(볼드) + 보조 라벨(예: “AI 추천 메뉴”) + 선택 시 체크 아이콘(간결).
- 보조 영역(옵션): 태그 칩 1~2개 또는 한 줄 설명 스니펫.
- 배경: 다크 패널 위 얕은 그라데이션(좌→우), 그림자 약하게 적용.
- 테두리: 기본 Neutral 라인, 선택 시 Primary/Accent 라인 + 은은한 글로우.

## 5) 상태 정의
- 기본: 중립 배경 + 얇은 라인.
- Hover/Focus: 라인 강조 + 살짝 상승(`-translate-y`), 그림자 강화.
- Active/Pressed: 눌림 효과(`scale-98`), 배경 어둡게.
- Selected: Primary 그라데이션/라인, 텍스트 컬러 반전.
- Disabled(로딩 중): 불투명도 60% + 커서 기본.

## 6) 상호작용/애니메이션
- 엔트리 애니메이션: 추천 도착 시 순차 fade/slide (`menu-card-enter`), 80~120ms 스텝.
- Hover: 40~80ms로 가벼운 모션, 그림자/라인 강조.
- 클릭 시: 잔잔한 scale 효과, 포커스 링 표시(접근성).

## 7) 반응형/스크롤
- 모바일: 1열, 좌우 패딩 축소, 배지/텍스트 크기 -1 step.
- 태블릿: 2열, 여백 표준.
- 데스크톱: 3열, `max-height` 240~280px 세로 스크롤 유지.

## 8) 접근성
- 키보드 탭 포커스 링 필수, 클릭 영역 전체 버튼화.
- 대비: 배경/텍스트 최소 4.5:1, 배지 대비 3:1 이상.
- 라이브 영역 없이도 정보 파악 가능하도록 배지/텍스트만으로 상태 전달.

## 9) 구현 체크리스트

- [ ] 카드 기본/hover/active/selected/disabled 상태 토큰 정의
- [ ] 좌측 컬러 스트립 + 소형 사각 랭크 태그 + 보조 라벨 추가
- [ ] 선택 상태 색/라인/텍스트 대비 및 체크 아이콘 처리
- [ ] 반응형 타이포·패딩 스케일 정의
- [ ] 포커스 링 및 접근성 대비 검증
